Bring Your Code to Life with Github Pages!

Everyone has heard of GitHub — it's a widely-loved platform where developers store, manage, and share their code. In essence, GitHub is an online repository that makes your projects accessible from anywhere with an internet connection. But today, we won’t be diving into the typical GitHub features. Instead, we'll explore a lesser-known but powerful tool — GitHub Pages.

GitHub Pages lets you turn your code into a live website, making it much more than just a repository of source code. Rather than just storing files, GitHub Pages hosts your project online, translating your code into a fully functional website. Even basic HTML files can be transformed into interactive pages that others can access just by visiting a link. This guide will walk you through the simple steps to set up your own GitHub Page — and yes, this very tutorial is brought to you using GitHub Pages!

Step 1: Sign Up for GitHub

If you don't have a GitHub account yet, head over to the GitHub signup page and create one. After signing up, log in to access the GitHub interface.

Step 2: Create a New Repository

Once you're logged in, the first step is to create a new repository. A repository (or "repo") is essentially a folder where you store your code. To create one, click the "+" button in the upper-right corner of the page, then select "New repository."

In the repository creation interface, you'll need to name your repo — this is the most important field. You can leave the other options at their default settings. Once you're done, scroll to the bottom and click Create repository.

Step 3: Upload Your HTML File

Now that your repository is created, you can upload files to it. GitHub allows several methods to add files, but for this tutorial, we’ll use the "Upload files" option.

To do this, you'll first need an HTML file ready on your local machine. Create a simple HTML file in any folder on your computer. Name it index.html — this is the default file GitHub Pages will look for when hosting your site. If you're unsure what to include, here's a basic HTML template to get you started:

<!DOCTYPE html>
<html>
<head>
    <title>My GitHub Page</title>
</head>
<body>
    <h1>Welcome to My GitHub Page</h1>
    <p>This is a simple demo of GitHub Pages in action.</p>
</body>
</html>

Save this file, and make sure it’s named index.html (if you don’t see the file extension, you may need to enable "Show file extensions" in your file explorer settings).

Back on GitHub, go to your repository and click Upload files. Select the index.html file from your local machine, and click Commit changes to upload it.

Step 4: Publish Your GitHub Page

Now for the magic. Go to the Settings tab at the top of your repository page. Scroll down to the Pages section on the left-hand sidebar.

In the Branch dropdown, select main (or the branch where your code is stored). Then click Save. After a few moments, GitHub will display a banner at the top of the page, notifying you that your site is live.

This banner will also include the link to your brand-new webpage. Click the link, and voilà — your code is live!

Final Thoughts

And that’s it! You've just created and hosted a live website using GitHub Pages. This is just the beginning, though. GitHub Pages offers a lot more flexibility and customization, allowing you to tweak your project as you see fit.

Explore further and see what else you can build. Until next time, happy coding!
